01 / 18
The big idea

It's a whole software team — in a box

One AI writing code alone is like a freelancer with no manager. Agent Loop is the opposite: a coordinated team, where each AI has a job and a supervisor makes sure the work is actually good before anyone calls it done.

📦 Agent Loop
🔍 Researcher 🗺️ Planner 🛠️ Builder 👀 Inspector ✅ Tester 🧑‍💼 Supervisor
🏗️

Think of it like a general contractor. You describe what you want — and a managed crew handles the research, planning, building, and inspection. You don't babysit every step.

02 / 18
Why it exists

Why not just ask one AI to "build it"?

Because no one checks the work. Here's the difference:

😵 One AI, going solo

  • Guesses what you really meant
  • Writes code with no one reviewing it
  • You discover the bugs later — the hard way
  • If it gets stuck, it just keeps churning

✅ Agent Loop

  • Agrees on the goal before building
  • A second AI inspects every change
  • Fixes problems automatically, then re-tests
  • Knows when it's stuck — and stops to tell you

How they coordinate

How the agents talk to each other

Here's the surprising part: the agents never chat live. Each one writes its work into a shared workspace, and the next one reads it before starting.

🔍 Researcher →📄→ 🗺️ Planner →📄→ 🛠️ Builder →📄→ 👀 Inspector
🗂️ One shared workspace — the single source of truth
📄 the map 📋 the checklist 🗺️ the plan ✅ the to-dos 📝 review notes 🧠 lessons learned

📝 Numbered notes

During reviews, feedback comes as numbered items — so nothing slips through, and the supervisor checks each one is resolved before moving on.

🆕 Fresh eyes on demand

Sometimes an agent gets only the documents, with no memory of the earlier chat — a deliberate, unbiased second opinion.

💡

Like a shared drive plus tracked comments: everyone works off the same files, every comment is numbered, and nothing's "done" until each one is ticked off.

Step 1 · Research

Look before you leap

Before planning anything, the Researcher reads through your existing code and writes a short map of how things work today.

🔍 Researcher → 🗺️ A map of your code → 👥 Shared with the whole team
🏗️

Like a surveyor walking the site before construction — so no one builds blind. It's done once, then reused by every step that follows.

Step 2 · Agree

Nail down what "done" means — first

A short back-and-forth clears up open questions, then the team writes a checklist of requirements. Every item is tracked, so nothing slips through later.

💬 Questions & answers →
📋 The checklist
  • Requirement 1
  • Requirement 2
  • Requirement 3
💡

Like a blueprint plus a punch-list: agree on it up front, then tick every item off at the end. The team can even debate the plan to pressure-test it.

Step 3 · Plan

🧩

Like a project board: one big project becomes small tickets, done in the right sequence — no one trips over unfinished work.

Step 4 · Build

Three ways to get the work done

📦

All at once

Hand the whole list to one builder. Simple and quick — best for small jobs.

🚶‍♂️🚶‍♀️

One by one

Finish each task fully before the next. Careful and tidy — best for step-by-step work.

⚡

Parallel teams

Independent tasks run at the same time. Fastest for big jobs with many moving parts.

Agent Loop works out the safe order — you just pick the style.

Self-correction

Failed the test? It tries again — by itself

When a test fails, Agent Loop feeds the exact feedback back in and retries automatically. No copy-pasting error messages back to it.

🛠️ Build → ✅ Test ✕ → 📝 Read the feedback → 🛠️ Build again → ✅ Test ✓
💡

Like a student who reads the teacher's red marks and revises — instead of handing in the same draft again.

13 / 18
Safety nets

It raises its hand instead of spinning

A good worker speaks up when they're stuck. Agent Loop watches for three warning signs:

🔁

Going in circles

Keeps making the same change over and over.

🧊

No progress

Nothing actually changed this round.

⏰

Taking too long

It's been working past a sensible time limit.

🧠

Remembers lessons

Writes down what it learned and applies it next time.

When stuck, it warns you or stops safely — rather than burning time (and money) going nowhere.

Right-sized process

A small fix or a big project? It scales to fit.

You don't run the whole machine for a one-line change. Agent Loop dials the process up or down to match the job in front of it.

⚡ Simple job

A small, clear change — low risk, one obvious path.

🛠️ Build→ ✅ Check

One assistant · minutes · light touch

e.g. fix a bug · tweak a label · add one small endpoint

🏗️ Not-so-simple job

A big feature with many moving parts and higher stakes.

🔍→ 📋→ 🗺️→ 🛠️→ 👀→ ✅→ 🔁

Several assistants · full paper trail · more checkpoints

e.g. billing · API redesign · data migration

💡

Same team, same tools — you just choose how much process the job deserves. The full pipeline is there when you need it, and out of the way when you don't.
